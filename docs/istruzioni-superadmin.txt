[COVER subtitle="Backup, ripristino e gestione di sistema"|role=Guida SuperAdmin|date=14/06/2026]

= 1. PANORAMICA =

  Il SuperAdmin e' un ruolo speciale che estende i privilegi
  dell'admin standard con operazioni di sistema:
    - Backup completo del database (download JSON)
    - Ripristino del database da backup (upload JSON)
    - Esportazione dello storico audit log (download JSON)

  A differenza dell'admin, il SuperAdmin NON e' visibile nelle
  liste pubbliche dell'applicazione (vedi sezione 3).

= 2. ACCESSO E PRIMO AVVIO =

  Al primo avvio (o dopo reset del DB), il server crea
  automaticamente un SuperAdmin di default:
    [INFO Email|superadmin@valbellunamotorsport.it]
    [INFO Password|superadmin123]

  Il login e' come qualsiasi altro utente:
    - Apri il browser all'indirizzo dell'applicazione
    - Inserisci email e password
    - Dopo il login, vedi la dashboard delle gare

  Per accedere alle funzioni SuperAdmin:
    - Vai su "Admin Dashboard" dal menu in alto
    - Le card "Backup Dati" e "Ripristina Backup" sono
      visibili SOLO a te
    - Nella pagina "Storico" vedi il pulsante "Esporta Storico"

  [WARNING] Cambia IMMEDIATAMENTE la password di default.
  Vai su "Impostazioni" (menu in alto, icona ingranaggio)
  per cambiare password o email.

= 2.5 TEMA SCURO / CHIARO =

  Clicca l'icona sole/luna in alto a destra per passare dalla
  modalita' chiara a quella scura. La preferenza viene salvata
  automaticamente. Al primo accesso, il tema segue le
  impostazioni del tuo sistema operativo.

= 3. COMPORTAMENTO INVISIBILE =

  Il SuperAdmin non compare in nessuna delle seguenti viste:
    - Lista membri (Admin Dashboard > Membri)
    - Partecipanti nelle pagine di dettaglio gara
    - Report e statistiche
    - Contatori della dashboard admin (membri attivi, ecc.)

  Solo il database e gli altri SuperAdmin sanno della tua
  esistenza. Gli admin standard e i membri non vedono alcuna
  traccia del SuperAdmin nell'applicazione.

= 4. BACKUP COMPLETO (ESPORTAZIONE JSON) =

  Scarica un file JSON contenente TUTTI i dati del database.

== 4.1 DAL BROWSER ==

    - Vai su "Admin Dashboard"
    - Clicca sulla card "Backup Dati"
    Oppure apri direttamente:
      [CODE] http://localhost:8000/races/admin/export

== 4.2 CONTENUTO DEL FILE JSON ==

  Il backup include, nell'ordine (rispettando le FK):
    - race_types: tipi gara (codice, descrizione)
    - users: tutti gli utenti (nome, email, ruolo, stato)
    - races: tutte le gare (date, descrizione, tipo, stato, ecc.)
    - participations: tutte le partecipazioni (utente, gara,
      stato, nota, macchina)
    - audit_logs: cronologia delle modifiche

  [TIP] Il file e' auto-consistente e puo' essere utilizzato per
  un ripristino completo (sezione 5).

= 5. RIPRISTINO DA BACKUP (IMPORTAZIONE JSON) =

  Carica un file JSON di backup per ripristinare i dati.

== 5.1 DAL BROWSER ==

    - Vai su "Admin Dashboard"
    - Clicca sulla card "Ripristina Backup"
    - Seleziona il file JSON dal tuo computer
    - Il ripristino parte automaticamente

== 5.2 COMPORTAMENTO ==

    - Il ripristino elimina TUTTI i dati esistenti e li
      sostituisce con quelli del backup
    - L'operazione avviene in una singola transazione SQL:
      se qualcosa va storto, tutto viene annullato e il
      database rimane invariato
    - Il file deve avere la stessa struttura dell'export
      generato dal sistema (sezione 4)

  [WARNING] Il ripristino sovrascrive irreversibilmente tutti i dati
  presenti. Effettua un backup preventivo prima di procedere.

= 6. STORICO AUDIT LOG =

  La pagina "Storico" mostra il registro cronologico di
  tutte le modifiche al sistema. E' accessibile solo ai
  SuperAdmin.

== 6.1 PAGINA WEB ==

    - Vai su "Storico" nel menu in alto
    - Vedrai 50 eventi per pagina in ordine cronologico
      inverso (dal piu' recente)
    - Puoi filtrare per:
        * Azione (LOGIN, CREATE, UPDATE, DELETE, ecc.)
        * Utente
        * Data (da / a)
        * Indirizzo IP
        * Descrizione gara
    - Usa la paginazione in fondo per navigare

== 6.2 ESPORTAZIONE JSON ==

    - Nella pagina "Storico", clicca "Esporta Storico"
    - Oppure apri direttamente:
      [CODE] http://localhost:8000/history/export
    - L'export rispetta gli STESSI filtri applicati nella
      pagina web (se non ci sono filtri, esporta tutto)
    - Scarica un file JSON senza limite di righe

== 6.3 CONTENUTO DEL FILE ==

  Ogni evento include:
    - Data e ora
    - Nome dell'utente che ha fatto la modifica
    - Descrizione della gara interessata
    - Campo modificato
    - Valore precedente e nuovo valore
    - Indirizzo IP
    - User agent del browser

= 7. RIGA DI COMANDO (CLI) =

  Esegui dalla root del progetto con virtualenv attivo e .env
  configurato. Il CLI crea le tabelle all'avvio ed esce con
  codice 1 in caso di errore. Dove indicato, --password e'
  opzionale: se omesso, prompt interattivo nascosto con doppia
  conferma (min 6 caratteri).

== 7.1 LISTA UTENTI ==

  [CODE] .venv/bin/python -m app.cli list-users

  [INFO Parametri|Nessuno]

  Mostra tutti gli utenti (inclusi SuperAdmin, nascosti in UI).

  [CODE] .venv/bin/python -m app.cli list-users

== 7.2 RESET PASSWORD ==

  [CODE] .venv/bin/python -m app.cli reset-password <email> [--password <password>]

  [INFO Posizionali|email (richiesto)]
  [INFO Opzionali|--password (se omesso, prompt interattivo)]

  [WARNING] Con --password la stringa e' in chiaro nella shell.

  [CODE] .venv/bin/python -m app.cli reset-password \
    superadmin@valbellunamotorsport.it
  [CODE] .venv/bin/python -m app.cli reset-password \
    mario.rossi@valbellunamotorsport.it --password nuovaPass99

== 7.3 PROMUOVI A SUPERADMIN ==

  [CODE] .venv/bin/python -m app.cli make-superadmin <email>

  [INFO Posizionali|email (richiesto)]

  [CODE] .venv/bin/python -m app.cli make-superadmin \
    admin@valbellunamotorsport.it

== 7.4 PROMUOVI A ADMIN ==

  [CODE] .venv/bin/python -m app.cli make-admin <email>

  [INFO Posizionali|email (richiesto)]

  [CODE] .venv/bin/python -m app.cli make-admin \
    mario.rossi@valbellunamotorsport.it

== 7.5 CREA NUOVO SUPERADMIN ==

  [CODE] .venv/bin/python -m app.cli create-superadmin <email> <nome> \
    [--password <password>]

  [INFO Posizionali|email (richiesto)]
  [INFO Posizionali|nome (richiesto, virgolette se con spazi)]
  [INFO Opzionali|--password (se omesso, prompt interattivo)]

  [CODE] .venv/bin/python -m app.cli create-superadmin \
    admin2@valbellunamotorsport.it "Admin Due" --password forte123

== 7.6 ELIMINA UTENTE ==

  [CODE] .venv/bin/python -m app.cli delete-user <email> [-y]

  [INFO Posizionali|email (richiesto)]
  [INFO Opzionali|-y (salta conferma)]

  [WARNING] Irreversibile. Tutti i dati dell'utente persi.

  [CODE] .venv/bin/python -m app.cli delete-user \
    vecchio.utente@example.com -y

== 7.7 CANCELLA LOG ==

  [CODE] .venv/bin/python -m app.cli clear-logs

  [INFO Parametri|Nessuno]

  [WARNING] Irreversibile. Backup prima di procedere.

= 8. GESTIONE ACCOUNT SUPERADMIN =

== 8.1 CREARE UN NUOVO SUPERADMIN ==

  Dal terminale:
  [CODE] .venv/bin/python -m app.cli create-superadmin \
    email@example.com "Nome Cognome"

  Oppure promuovendo un utente esistente:
  [CODE] .venv/bin/python -m app.cli make-superadmin email@example.com

== 8.2 RESETTARE LA PASSWORD ==

  Dal terminale:
  [CODE] .venv/bin/python -m app.cli reset-password \
    superadmin@valbellunamotorsport.it

  Ti verra' chiesta la nuova password (due volte per
  conferma). In alternativa, con --password:

  [CODE] .venv/bin/python -m app.cli reset-password \
    superadmin@valbellunamotorsport.it --password nuova123

== 8.3 ELIMINARE UN SUPERADMIN ==

  Dal terminale:
  [CODE] .venv/bin/python -m app.cli delete-user \
    superadmin@valbellunamotorsport.it

  [WARNING] L'eliminazione e' irreversibile. Assicurati che esista
  almeno un altro SuperAdmin prima di eliminare l'ultimo — senza
  SuperAdmin non e' possibile effettuare backup/ripristino
  dall'interfaccia web.

== 8.4 CAMBIARE PASSWORD/EMAIL DAL BROWSER ==

  Vai su "Impostazioni" (menu in alto, icona ingranaggio)
  per cambiare la tua password o email. Funziona come per
  qualsiasi altro utente.

= 9. NOTE E AVVERTENZE =

    - Il SuperAdmin di default viene ricreato automaticamente
      dopo un reset del database
    - I backup JSON NON contengono password in chiaro (solo hash)
    - Conserva i backup in un luogo sicuro: contengono dati
      personali di tutti gli utenti
    - Non eliminare l'ultimo SuperAdmin: non ci sarebbe piu'
      modo di fare backup/ripristino dall'interfaccia web
    - I token di registrazione sono persistenti nel database:
      non vengono persi al riavvio del server
    - Ogni token ha una scadenza configurabile (minuti, ore,
      giorni) o puo' essere senza scadenza
